## 📝 Der perfekte IT-Lebenslauf & Anschreiben

Vergiss den 3-seitigen Fließtext. In der IT wird gescannt, nicht gelesen.

<details open>
<summary><b>Der Lebenslauf (CV): Kompakt und Skills-orientiert</b></summary>

- **Länge:** Maximal 1 bis 2 Seiten (als sauberes PDF). Nutze Vorlagen von *Canva* oder *Novoresume* für ein modernes, klares Design (zwei Spalten).
- **Die Skill-Matrix (WICHTIG!):** Baue sofort auf der ersten Seite (z.B. in der Seitenspalte) eine Skill-Liste ein. Teile sie ehrlich auf:
  - *Fortgeschritten (Daily Business):* C#, ASP.NET Core, Git, HTML/CSS
  - *Grundlagen:* SQL, Docker, TypeScript, Azure
- **Vorherige Berufe (Der Umschüler-Vorteil):** Verstecke deine Vergangenheit nicht! Hast du vorher im Einzelhandel gearbeitet? Dann schreibe dazu: *"Soft-Skills: Extreme Kundenorientierung, Stressresistenz, Beschwerdemanagement"*. Mach aus dem "Nachteil" deines Alters den Vorteil deiner echten Lebenserfahrung!
</details>

<details open>
<summary><b>Das Anschreiben (Motivation statt Wiederholung)</b></summary>

- **Niemals:** "Hiermit bewerbe ich mich als... ich habe Ihren Artikel gelesen...". Keine Standardfloskeln!
- **Der Einstieg:** Komm direkt zur Sache. *"Guten Tag Herr X, Sie suchen einen Junior .NET Entwickler, der nicht nur Code tippt, sondern auch die Geschäftslogik dahinter versteht. Als Umschüler bringe ich genau diese Reife mit."*
- **Die Begründung:** Erzähle nicht deinen Lebenslauf nach (den lesen sie ja eh). Erkläre, *warum* du in der Freizeit an einem bestimmten GitHub-Projekt baust oder *welches Problem* du in der Firma lösen willst!
</details>

## 📸 Das Business-Foto (Keine Selfies!)

Ein schlechtes Foto katapultiert dich sofort in den Papierkorb.

<details open>
<summary><b>Regeln für das perfekte Bild</b></summary>

1. **Kein Handy-Selfie vor grauer Raufasertapete!** 
2. **Die Kleidung:** Dresscode in der IT ist locker. Ein sauberes, unifarbenes T-Shirt mit Sakko (Männer) oder eine schlichte Bluse/Hemd reichen oft schon. Niemand erwartet einen Smoking (es sei denn, du bewirbst dich bei einer klassischen Bank).
3. **Der Hintergrund:** Lass es vom Fotografen machen (kostet 30-50 Euro, eine Investition, die sich sofort lohnt). Nutze moderne, unscharfe Hintergründe (z.B. ein modernes Büro oder eine Backsteinwand, kein grelles Studio-Blau).
4. **Ausstrahlung:** Ein sympathisches, waches Lächeln ist wichtiger als "cool" auszusehen.
</details>

## 🌐 LinkedIn, Xing & Indeed (Dein digitales Schaufenster)

Jobsuche in der IT funktioniert heute oft umgekehrt: Du wirst auf LinkedIn von "Headhuntern" oder Recruitern gefunden (*Inbound Recruiting*).

<details open>
<summary><b>LinkedIn: Das Must-Have</b></summary>

- **Slogan (Headline):** Nicht "Umschüler auf Jobsuche". Schreib konkret, was du machst: *"Junior C# / .NET Developer | ASP.NET Core & Azure"*
- **Profilbild & Banner:** Nutze dein professionelles Business-Foto. Erstelle in Canva ein passendes Banner (z.B. ein dezentes Code-Muster).
- **Die Info (About-Section):** Schreibe 3-4 Sätze über deine Reise als Umschüler und für welche Tech-Stacks du brennst. Erwähne unbedingt deinen GitHub-Link!
- **Aktivität:** Kommentiere gelegentlich Beiträge von Firmen, die dich interessieren. Teile kleine Meilensteine (z.B. "Mein erstes ASP.NET MVC Projekt ist live!").
- *"Open to Work":* Schalte im Profil den Status auf *"Offen für Jobangebote"*, damit Recruiter dich in ihren Suchtools finden.
</details>

<details open>
<summary><b>Xing & Indeed</b></summary>

- **Xing:** Reicht heutzutage im deutschsprachigen Raum aus, wird aber von LinkedIn stark verdrängt. Baue dort einen rudimentären Skill-Katalog auf und setze die Job-Wünsche auf aktiv.
- **Indeed Profil:** Wenn du über Indeed suchst, lade dort deinen sauber optimierten PDF-Lebenslauf hoch. Nutze auf Indeed in den Einstellungen Keywords (C#, SQL, Junior), damit dein Profil Personalern vorgeschlagen wird.
</details>

## 🐱 Das GitHub Profil & Portfolio (Dein echtes Zeugnis)

Für Entwickler (FIAE) ist GitHub der eigentliche Lebenslauf. Ein Senior-Entwickler, der deine Bewerbung prüft, liest im Schnitt keine 10 Sekunden in deinem Word-Dokument. Er klickt auf deinen GitHub-Link und schaut sich deinen Code an.

<details open>
<summary><b>1. Das GitHub Profil-README (Der Wow-Faktor)</b></summary>

Wusstest du, dass du eine "Startseite" für dein GitHub-Profil bauen kannst?
1. Erstelle ein neues, völlig leeres Repository auf GitHub, das **exakt den gleichen Namen hat wie dein GitHub-Benutzername** (z.B. `github.com/MaxMuster/MaxMuster`).
2. GitHub markiert dies als `✨ special ✨` Repository.
3. Erstelle eine `README.md` darin. Diese wird ab sofort riesengroß auf deiner Profil-Startseite gerendert!
4. **Was kommt da rein?** Nutze [Profil-Generatoren](https://rahuldkjain.github.io/gh-profile-readme-generator/), um Icons deiner bekannten Programmiersprachen, deine LinkedIn-Kontaktdaten und dynamische Badges (z.B. "Bisherige Commits") einzubinden.
</details>

<details open>
<summary><b>2. Die Portfolio-Projekte (Qualität über Quantität)</b></summary>

Niemand will 50 "Hello World"-Konsolenanwendungen sehen. 
- **Was zählt:** Zeige 2 bis 3 echte, funktionierende Projekte.
- **Vollständigkeit:** Jedes dieser Repositories braucht eine *eigene* `README.md`! Wenn ich auf dein "ShoppingList"-Projekt klicke und nicht sehe, was es macht, schließe ich den Tab.
  - Mach einen **Screenshot** der fertigen UI und füge ihn in die README ein.
  - Schreibe in die README, *wie* man das Projekt startet (`dotnet run`).
  - Zähle die verwendeten Tech-Stacks auf (C#, MVC, Entity Framework, Tailwind).
- **Die Commits:** Ein Projekt, das am 1. Februar erstellt wurde und wo "Initial Commit" den gesamten Code hochlädt, wirkt kopiert. Zeige durch viele kleine Commits über Wochen (`feat: login added`, `fix: css bug`), dass du *wirklich* daran gearbeitet hast.
</details>
